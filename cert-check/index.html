Tentang SSL/TLS

Algoritma

SSL (Secure Sockets Layer) dan TLS (Transport Layer Security) adalah protokol keamanan yang dirancang untuk memberikan komunikasi aman melalui jaringan komputer. TLS adalah pengganti SSL yang lebih baru dan lebih aman.

Protokol ini mengenkripsi data yang dikirimkan antara browser dan server, sehingga informasi sensitif seperti nomor kartu kredit, nama pengguna, dan kata sandi tetap privat dan aman.

  • SSL/TLS menggunakan kombinasi kriptografi asimetris dan simetris untuk mengamankan koneksi
  • Proses "handshake" SSL/TLS mengotentikasi server dan menetapkan parameter enkripsi
  • Sertifikat digital digunakan untuk memverifikasi identitas server
  • TLS 1.3 adalah versi terbaru yang menawarkan keamanan dan performa yang lebih baik

Saat Anda mengunjungi situs web dengan HTTPS, browser Anda melakukan handshake SSL/TLS dengan server untuk memastikan koneksi aman sebelum mengirimkan data apa pun.

Sertifikat SSL/TLS adalah file data yang mengikat kunci kriptografis ke detail organisasi. Saat diinstal pada server web, sertifikat memungkinkan situs untuk menggunakan koneksi HTTPS dan memastikan keamanan data.

  • Sertifikat diterbitkan oleh Otoritas Sertifikat (CA) tepercaya seperti Let's Encrypt, DigiCert, atau GlobalSign
  • Setiap sertifikat memiliki periode validitas, biasanya antara 90 hari hingga 2 tahun
  • Sertifikat berisi informasi seperti nama domain, penerbit, tanggal kedaluwarsa, dan kunci publik
  • Browser memeriksa validitas sertifikat setiap kali Anda membuat koneksi HTTPS

Ada beberapa jenis sertifikat SSL:

  • Domain Validated (DV): Hanya memverifikasi kepemilikan domain, tingkat validasi paling dasar
  • Organization Validated (OV): Memverifikasi kepemilikan domain dan informasi organisasi
  • Extended Validation (EV): Tingkat validasi tertinggi dengan pemeriksaan menyeluruh terhadap organisasi
  • Wildcard: Melindungi domain dan semua subdomainnya (*.example.com)
  • Multi-Domain (SAN): Melindungi beberapa domain dengan satu sertifikat

SSL/TLS menggunakan berbagai algoritma kriptografi untuk mengamankan komunikasi. Keamanan koneksi bergantung pada kekuatan algoritma yang digunakan.

Tipe Algoritma Keamanan Keterangan
Protokol TLS 1.3 Tinggi Versi terbaru, menghapus algoritma tidak aman, performa lebih baik
Protokol TLS 1.2 Tinggi Masih aman dengan konfigurasi cipher suite yang tepat
Protokol TLS 1.0/1.1 Rendah Mengandung kerentanan, tidak disarankan untuk digunakan
Protokol SSL 3.0/2.0 Tidak Aman Usang dan rentan terhadap serangan seperti POODLE
Cipher Suite AES-GCM Tinggi Enkripsi simetris dengan mode authenticated encryption
Cipher Suite AES-CBC Sedang Masih aman tetapi rentan terhadap serangan seperti BEAST
Cipher Suite RC4, 3DES, DES Tidak Aman Algoritma usang dengan kerentanan serius
Tanda Tangan ECDSA Tinggi Kriptografi elliptic curve, lebih efisien dan aman
Tanda Tangan RSA Sedang Masih aman dengan kunci yang cukup panjang (2048+ bit)
Tanda Tangan MD5, SHA-1 Tidak Aman Algoritma hash yang rentan terhadap collision attacks
