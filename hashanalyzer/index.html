Informasi Hash

Tentang Hash
Keamanan
Perbandingan

Hashing adalah proses mengubah data menjadi nilai tetap panjang menggunakan algoritma matematika. Hash berfungsi sebagai one-way function, artinya mudah untuk menghitung hash dari input, tetapi sangat sulit untuk menentukan input dari hash.

Hash banyak digunakan dalam keamanan komputer untuk menyimpan password, memverifikasi integritas data, dan tanda tangan digital. Ketika Anda membuat password di situs web, biasanya password tersebut di-hash sebelum disimpan di database.

  • Hash selalu menghasilkan output dengan panjang yang sama, terlepas dari panjang input
  • Perubahan kecil pada input akan menghasilkan perubahan besar pada output hash
  • Hash adalah fungsi satu arah, tidak dapat dibalik untuk mendapatkan input asli
  • Dua input yang berbeda seharusnya tidak menghasilkan hash yang sama (collision resistance)

Keamanan hash sangat penting dalam melindungi data sensitif seperti password. Beberapa algoritma hash lama seperti MD5 dan SHA-1 dianggap tidak aman untuk tujuan keamanan karena kerentanan terhadap collision attacks.

Untuk meningkatkan keamanan password, banyak sistem menggunakan teknik "salting" di mana string acak (salt) ditambahkan ke password sebelum di-hash. Ini mencegah penggunaan rainbow table dan membuat serangan dictionary lebih sulit.

  • Gunakan algoritma hash modern seperti SHA-256, SHA-3, atau bcrypt untuk keamanan optimal
  • Selalu gunakan salt unik untuk setiap password untuk mencegah serangan precomputation
  • Pertimbangkan menggunakan fungsi derivasi kunci seperti PBKDF2, scrypt, atau Argon2 untuk password
  • Hindari menggunakan hash yang sama untuk tujuan berbeda dalam sistem yang sama

Berikut adalah perbandingan beberapa algoritma hash yang umum digunakan:

Algoritma Panjang Output Keamanan Kecepatan
MD5 128 bit (32 hex) Rendah (Vulnerable) Sangat Cepat
SHA-1 160 bit (40 hex) Rendah (Vulnerable) Cepat
SHA-256 256 bit (64 hex) Sedang Sedang
SHA-512 512 bit (128 hex) Tinggi Sedang
bcrypt Variable Tinggi Lambat (Designed)
Argon2 Variable Sangat Tinggi Dapat Disesuaikan

Algoritma seperti bcrypt dan Argon2 dirancang khusus untuk hashing password dan memiliki fitur seperti work factor yang dapat disesuaikan untuk membuat proses hashing lebih lambat dan lebih sulit untuk di-crack.
